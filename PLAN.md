# ai-model-benchmarks — Plan

## Vision

The first open-source project combining benchmarks + pricing + routing + capabilities + freshness + portal in one place.
The niche is open: none of the 30+ researched repos does this combination.

## Competitive landscape

| Project               | Stars | What it does                                         | What it lacks                              |
| --------------------- | ----- | ---------------------------------------------------- | ------------------------------------------ |
| chinese-llm-benchmark | 5.7K  | Markdown tables, 359+ models, updates every 1-7 days | No pricing, no routing, no portal, no JSON |
| Wu Long Arena Archive | —     | Daily GitHub Actions, JSON snapshots, REST API       | Arena ELO only, no pricing/routing         |
| DailyBench            | 17    | 4x/day via GH Actions, HELMLite                      | Only 3 models, no pricing, costs $5/day    |
| LiteLLM               | 41K+  | Unified API, auto-updated pricing, routing           | Not a benchmark data product — it's an SDK |
| RouteLLM              | —     | Router model (trains ML)                             | Research project, not a data reference     |
| OpenCompass           | 6.8K  | CompassRank visualization                            | Only its own evals, not an aggregator      |
| Artificial Analysis   | —     | Independent measurements + Stirrup harness           | Closed-source, not community-driven        |

**Our gap:** benchmarks + pricing + routing + capabilities + freshness + portal + auto-update. Nobody combines all of this.

## Architecture

```
OpenRouter API  ──→  sync_capabilities.py  ──→  data/models.json (capabilities)
                ──→  fetch_openrouter_prices.py  ──→  data/models.json (prices)
                                                      data/pricing.json
Provider docs   ──→  manual_capabilities.json  ──→  data/models.json (manual merge)

data/*.json  ──→  generate_portal.py  ──→  index.html (GitHub Pages)
             ──→  generate_md.py      ──→  MODEL_BENCHMARKS.md
             ──→  validate.py         ──→  CI checks

Daily CI: sync_capabilities → fetch_prices → generate_portal → auto-commit
```

## Decisions

1. **MD primary, JSON backing** — humans read MD, automation reads JSON
2. **GitHub Pages portal** — not HF Spaces (full control)
3. **PR contributions with source link** — not email (better for open-source)
4. **Daily price + capability updates** — sufficient (not 4x/day like DailyBench)
5. **Per-score dates** — our killer feature, nobody else does this
6. **Two-layer capabilities** — auto (OpenRouter, 97 models) + manual (provider docs, 15 top models)
7. **Anti-staleness by design** — every data layer has auto-update or staleness detection, nothing silent
